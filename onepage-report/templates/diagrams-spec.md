# diagrams.md 圖表規格（精簡版）

> **設計原則**：只寫關鍵資訊，視覺細節留給 Phase 6 渲染時決定

## 檔案格式

````markdown
# 圖表集

## 主圖：{圖表標題}

- **類型**：{before_after | platform_compare | flow | timeline | architecture}
- **說明**：{一句話說明這張圖要表達什麼}

### 節點列表
1. {節點名稱}：{簡短說明}
2. {節點名稱}：{簡短說明}
3. ...

### 重點標註
- {需要特別強調的內容}

---

## 附錄圖 1：{圖表標題}

- **類型**：{類型}
- **說明**：{說明}

### 節點列表
1. ...

---

## 附錄圖 2：...
````

---

## 圖表類型簡介

| 類型 | 適用場景 | 優先級 |
|------|---------|--------|
| `before_after` | 改善效果、問題解決前後對比 | 主圖首選 |
| `platform_compare` | PC vs 手機、不同平台機制對照 | 附錄常用 |
| `flow` | 單一流程、步驟說明、機制運作 | 附錄常用 |
| `timeline` | 延遲分析、時序關係 | 附錄常用 |
| `architecture` | 系統架構、模組關係、改動位置 | 附錄必須 |

---

## 精簡原則

### ✓ 要寫的內容

| 項目 | 說明 |
|------|------|
| 節點名稱 | 每個節點的名稱 |
| 節點說明 | 1 行簡短說明 |
| 節點順序 | 流程/時間順序 |
| 重點標註 | 需要強調的差異或問題點 |
| 數據（如有） | 時間、百分比等關鍵數據 |

### ✗ 不要寫的內容

| 項目 | 原因 |
|------|------|
| 顏色規格 | Phase 6 會根據語意自動決定 |
| 箭頭樣式 | Phase 6 會統一處理 |
| 子節點細節 | 會讓規格過長，Phase 6 會補足 |
| 視覺排版 | Yoga Layout 自動計算 |
| 尺寸像素 | yoga_converter 自動計算，寫了也無效 |

---

## 範例

```markdown
## 主圖：Anti-Lag SDK 導入前後延遲對比

- **類型**：before_after
- **說明**：展示導入 SDK 前後，Frame Queue 問題如何被解決

### 節點列表

**改善前：**
1. 觸控事件：T=0
2. 輸入採樣：可能採到過時輸入
3. CPU 處理：產生 Frame Queue
4. Frame Queue 堆積：2-3 幀排隊（主要延遲來源）
5. GPU 渲染：渲染舊幀
6. 螢幕顯示：顯示過時畫面

**改善後：**
1. 觸控事件：T=0
2. SDK 同步點：等待 GPU 準備好
3. 輸入採樣：最佳時機採樣
4. CPU+GPU 同步：Frame Queue ≈ 0
5. 螢幕顯示：顯示最新畫面

### 重點標註
- Frame Queue 堆積是主要延遲來源
- SDK 同步點消除排隊現象
- 效果：延遲降低 5%+
```
