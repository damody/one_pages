## 尺寸規格

| 位置 | 尺寸（像素） | 說明 |
|------|-------------|------|
| 主圖 | 動態（見下表） | 根據內容量決定 |
| 附錄單圖 | 1056×461 | 附錄頁只有一張圖時 |
| 附錄兩圖 | 1152×202 | 附錄頁有兩張圖時 |
| 附錄 2x2 | 605×202 | 附錄頁有 3-4 張圖時 |

### 主圖尺寸選擇

| 主投影片內容量 | 主圖尺寸 |
|---------------|---------|
| 少（≤3 區塊） | 960×480 |
| 中（4 區塊） | 800×400 |
| 多（≥5 區塊） | 640×320 |

## 圖表類型簡介

| 類型 | 適用場景 | 優先級 |
|------|---------|--------|
| `before_after` | 改善效果、問題解決前後對比 | 主圖首選 |
| `platform_compare` | PC vs 手機、不同平台機制對照 | 附錄常用 |
| `flow` | 單一流程、步驟說明、機制運作 | 附錄常用 |
| `timeline` | 延遲分析、時序關係 | 附錄常用 |
| `architecture` | 系統架構、模組關係、改動位置 | 附錄必須 |

**⚠️ 各類型的詳細規範：** `Read {skill_dir}/templates/appendix/diagrams-spec-types.md`

## 精簡原則

### ✓ 要寫的內容

| 項目 | 說明 |
|------|------|
| 節點名稱 | 每個節點的名稱 |
| 節點說明 | 1 行簡短說明 |
| 節點順序 | 流程/時間順序 |
| 重點標註 | 需要強調的差異或問題點 |
| 數據（如有） | 時間、百分比等關鍵數據 |

### ✗ 不要寫的內容

| 項目 | 原因 |
|------|------|
| 顏色規格 | Phase 6 會根據語意自動決定 |
| 箭頭樣式 | Phase 6 會統一處理 |
| 子節點細節 | 會讓規格過長，Phase 6 會補足 |
| 視覺排版 | Yoga Layout 自動計算 |

## 範例（精簡版）

```markdown
## 主圖：Anti-Lag SDK 導入前後延遲對比

- **類型**：before_after
- **說明**：展示導入 SDK 前後，Frame Queue 問題如何被解決
- **尺寸**：800x400

### 節點列表

**改善前（左側）：**
1. 觸控事件：T=0
2. 輸入採樣：可能採到過時輸入
3. CPU 處理：產生 Frame Queue
4. Frame Queue 堆積：2-3 幀排隊（主要延遲來源）
5. GPU 渲染：渲染舊幀
6. 螢幕顯示：顯示過時畫面

**改善後（右側）：**
1. 觸控事件：T=0
2. SDK 同步點：等待 GPU 準備好
3. 輸入採樣：最佳時機採樣
4. CPU+GPU 同步：Frame Queue ≈ 0
5. 螢幕顯示：顯示最新畫面

### 重點標註
- 左側用紅色標示 Frame Queue 堆積
- 右側用綠色標示 SDK 同步點
- 中間箭頭標註「效果：延遲降低 5%+」
```
